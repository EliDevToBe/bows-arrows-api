    Introduction

    Welcome to the Bow's Arrows API, which is about ... archery ! üòÆ
    This API is still in development and aim to be as exhaustive as possible.
    Currently, this API is in a v1.0, you can see some of futur features in the Roadmap section.

    You can only access the Bow's Arrows API using the HTTP method    GET

    Base URL

    Base URL response

    {
  "brands": "https://bows-arrows-api.vercel.app/api/brands",
  "arrows": "https://bows-arrows-api.vercel.app/api/arrows"
}
    GET      https://bows-arrows-api.vercel.app/api

    The Base URL will greet you with the main endpoints available at the moment.
    These are Brands and Arrows for the moment.

    Arrows

    Any Arrow Object

    {
  "id": 123456,
  "name": "Some model name",
  "brand": {
      "name": "Big Brand",
      "url": "link-to-the-brand"
  },
  "discipline": "target or hunting",
  "material": {
      "name": "gold",
      "quality": "24k"
  },
  "tolerances": {
      "gpi": 0.002,
      "straightness": 0.004,
      "spine": null
  },
  "spinesUrl": "link-to-all-spines-available"
}

    This is the endpoint to retrieve all arrows in a JSON format.
    An arrow is considered an object and all Arrow Object follows the arrow schema.

    Arrow Schema

    Key Type Description
    id int Arrow id
    name string Model name of the arrow
    brand object Name and endpoint of the brand
    discipline string Indicating for which discipline the arrow is best used
    material object Name and Quality of the material
    tolerances object About Grain per inch, Straightness, Spine
    spinesUrl string Endpoint for retrieving all spines for this arrow

    This is the endpoint to retrieve an array of Arrow Object following the Arrow Schema

    GET      https://bows-arrows-api.vercel.app/api/arrows

    Get a Specific arrow

    Response using <BaseUrl>/arrows/182

    {
  "id": 182,
  "name": "tenpro",
  "brand": {
      "name": "fivics usa",
      "url": "https://bows-arrows-api.vercel.app/api/brands/8"
  },
  "discipline": "target",
  "material": {
      "name": "carbon",
      "quality": "30 ton"
  },
  "tolerances": {
      "gpi": null,
      "straightness": 0.002,
      "spine": null
  },
  "spinesUrl": "https://bows-arrows-api.vercel.app/api/arrows/182/spines"
}

    If you want to retrieve a specific arrow, an id parameter is required.

    GET      https://bows-arrows-api.vercel.app/api/arrows/<ID>

    URL Parameter

    Parameter Description
    ID The required id to retrieve

    Get All spines associated

    Simply adding /spines at the end of a specific arrow retrieves all the spines possible associated.

    GET      https://bows-arrows-api.vercel.app/api/arrows/<ID>/spines

    Errors

    What to expect when not using a GET request

    {
  "oops": "There is nothing here.",
  "method": "PUT",
  "message": "Method Not Allowed"
}

    When accessing the API with another HTTP method other than GET, you will receive a 405 code error with a visible JSON summarizing what went wrong.

    The Bow's Arrows API uses the following error codes:

    Error Code Meaning
    400 Bad Request -> Your request is invalid.
    405 Method Not Allowed -> You tried to access the API with an invalid method.
    500 Internal Server Error -> We had a problem with our server. Try again later.
